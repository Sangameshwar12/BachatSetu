# Development Roadmap

This roadmap is designed for an enterprise-grade startup launch while preserving speed. Dates should be assigned after team size, funding runway, compliance requirements, and launch market are confirmed.

## Phase 0: Foundation

Goals:

- Finalize architecture
- Finalize documentation
- Set up repository
- Establish engineering workflow
- Define first product scope

Deliverables:

- Repository foundation
- Architecture documentation
- Coding and security standards
- Initial product requirements
- UX wireframes
- Compliance review plan

## Phase 1: Platform Core

Goals:

- Build shared SaaS foundation
- Implement identity, tenants, roles, audit, and configuration
- Establish CI/CD and cloud baseline

Deliverables:

- Backend project initialized
- Mobile project initialized
- Admin portal initialized
- PostgreSQL and Redis local development
- Flyway baseline
- Authentication foundation
- Tenant and role model
- Audit framework
- Dev environment deployment

## Phase 2: Bhishi MVP

Goals:

- Support core Bhishi group management
- Support contribution tracking
- Support member-facing mobile experience
- Support admin operations

Deliverables:

- Bhishi group setup
- Member management
- Contribution schedules
- Contribution status
- Basic reminders
- Admin group oversight
- Core reports

## Phase 3: Payments and Ledger

Goals:

- Integrate payment provider
- Implement ledger-backed financial records
- Add reconciliation and operational visibility

Deliverables:

- Payment initiation
- Provider webhook handling
- Idempotency
- Ledger entries
- Reconciliation jobs
- Payment status reporting
- Support workflows for payment mismatches

## Phase 4: Private Beta

Goals:

- Launch with controlled user groups
- Observe real behavior
- Harden support and operations

Deliverables:

- Beta onboarding
- Production-like staging
- Monitoring dashboards
- Incident runbooks
- Feedback collection
- Security review
- Performance baseline

## Phase 5: Public Launch

Goals:

- Launch production platform
- Support growth safely
- Establish reliable release cadence

Deliverables:

- Production AWS environment
- Production deployment pipeline
- Mobile store release process
- Customer support process
- Backup and restore validation
- Security testing signoff
- Launch analytics

## Phase 6: Expansion Modules

Goals:

- Extend from Bhishi into broader community collection use cases
- Reuse platform primitives

Candidate modules:

- Women's Self Help Groups
- Society Maintenance
- Apartment Collection
- Temple Collection
- Festival Collection
- Office Contribution
- Wedding Collection
- Travel Saving Groups
- NGO Collection
- Community Funds
